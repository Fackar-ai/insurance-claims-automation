# Insurance Claims Automation System

## 🚀 Overview

This project demonstrates an end-to-end automation system for processing insurance claims using RPA, Python, APIs, and AI.

The solution automates document retrieval, validation, classification, and reporting across multiple platforms, significantly reducing manual workload.

---

## 📊 Business Impact

* Processes ~20,000 claims per year (~55/day)
* Reduced manual work by **~95%**
* Replaced **3–4 operational roles**
* Processing time reduced from **3–5 minutes → ~45 seconds per claim**

---

## ⚙️ Key Features

* Multi-source data extraction (Web platform, SFTP, OneDrive)
* AI-based document processing (Gemini API → JSON structured data)
* Automated validation and classification (new vs annex claims)
* Cross-system integration (Prometeo, external APIs)
* Automated notifications (SMS & email)
* Centralized reporting (Google Sheets)

---

## 🧩 Architecture (Simplified Flow)

1. Extract claims from web platform
2. Retrieve documents (SFTP / platform attachments)
3. Send documents to AI for structured data extraction
4. Validate data against external systems
5. Classify claim type (new or annex)
6. Store results in Google Sheets
7. Trigger notifications and closure workflows

---
